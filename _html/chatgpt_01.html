나의 말:
vscode에서 markdown 파일은 prettier가 적용되지 않도록 하려면?
ChatGPT의 말:

🧙🏾‍♂️: Markdown 파일에 Prettier가 자동으로 적용되지 않도록 설정하려면 .prettierignore 파일을 사용하거나 VS Code에서 특정 파일에 대해 Prettier를 비활성화할 수 있어요! 🛠️

💡 방법 1: .prettierignore 파일 사용
프로젝트 루트에 .prettierignore 파일을 생성하고, 아래 내용을 추가하세요. 

*.md

이렇게 하면 모든 Markdown 파일이 Prettier의 포맷팅 대상에서 제외됩니다.

💡 방법 2: VS Code에서 파일별로 설정
VS Code의 settings.json에 아래와 같이 추가할 수 있습니다. 

json
"[markdown]": {
  "editor.defaultFormatter": null
}

이렇게 하면 Markdown 파일에 대해 Prettier가 동작하지 않아요.

혹은, 특정 프로젝트에서만 적용하고 싶다면, .vscode/settings.json에 추가하세요.

🧙🏾‍♂️: 어떤 방식이 더 편리할 것 같나요? 😊

나의 말:
방법 2를 사용했는데, markdown 파일을 저장하면 '_'가 '\_'로 자동 변경되고, 줄바꿈도 여러 개가 2개로 바뀌네요.
ChatGPT의 말:

🧙🏾‍♂️: 해당 현상은 VS Code의 다른 확장 프로그램(예: Markdown All in One)이나 기본 Markdown 설정에서 자동 포맷팅이 이루어지고 있기 때문이에요! ✨

해결 방법:

1. Markdown All in One 확장 설정 변경

Markdown All in One이 설치되어 있다면, 이 확장에서 줄바꿈이나 특수 문자 이스케이프 처리를 합니다. 설정을 조정해보세요.

  • settings.json에 아래를 추가합니다. 
json
"markdown.extension.syntax.decorations": false,
"markdown.extension.print.onFileSave": false,
"markdown.extension.orderedList.autoRenumber": false,
"markdown.extension.list.indentationSize": "inherit",
"markdown.extension.italic.indicator": "*"
  • 특히, _가 \_로 바뀌는 것은 markdown.extension.syntax.decorations가 원인일 수 있어요.

2. 포맷터 비활성화

VS Code 기본 설정에서 Markdown에 대한 자동 포맷팅을 완전히 비활성화할 수도 있습니다. 

json
"[markdown]": {
  "editor.formatOnSave": false,
  "editor.formatOnPaste": false
}

3. Prettier 외 포맷터 확인

  • VS Code 명령어 팔레트에서 Format Document를 실행하고, 어떤 포맷터가 동작하는지 확인해보세요.
  • Prettier가 아니라 다른 확장이 포맷터로 동작하고 있다면, settings.json에서 해당 포맷터를 비활성화합니다. 
json
"[markdown]": {
  "editor.defaultFormatter": "esbenp.prettier-vscode"
}
  • 위처럼 설정해 특정 포맷터만 동작하도록 강제할 수 있습니다.

🧙🏾‍♂️: 설정 수정 후에도 문제가 지속되면 어떤 확장들이 활성화되어 있는지 알려주세요! 😊
